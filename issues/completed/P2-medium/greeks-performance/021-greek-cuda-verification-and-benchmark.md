## Summary

Verify CUDA Greek computation end to end and produce a benchmark before removing
legacy scalar/debug Greek paths or simplifying CLI backend options.

## Why It Matters

GPU acceleration should be optional and correct. Removing fallback/debug paths
before CUDA is verified could reduce debuggability or break non-GPU machines.

## Scope

In scope:

- Add or run GPU tests when CuPy and NVIDIA GPU are available.
- Produce benchmark document with CPU, scalar reference, and CUDA timings.
- Record hardware context and breakeven rows.
- Simplify CLI only after benchmark and verification gates pass.

Out of scope:

- Making CUDA a base dependency.
- Requiring GPU in default CI.

## Acceptance Criteria

- [x] CUDA backend matches NumPy backend within tolerance.
- [x] Invalid-row NaN behavior matches CPU.
- [x] Benchmark document is generated by script, not manual copy-paste.
- [x] CLI changes preserve CPU default and GPU opt-in behavior.
- [x] Scalar reference function remains available for tests/net Greeks where needed.

## Evidence Required

- GPU test output or skip reason
- generated benchmark document
- benchmark script

### Benchmark summary (RTX 3080, best-of-5, black76)

| Rows | numpy (ms) | cuda (ms) | cuda vs numpy |
|-----:|-----------:|----------:|--------------:|
| 1,000 | 0.155 | 1.639 | 0.1× |
| 10,000 | 0.988 | 2.093 | 0.5× |
| 100,000 | 13.011 | 6.195 | 2.1× |
| 1,000,000 | 175.582 | 88.094 | 2.0× |
| 5,000,000 | 1188.567 | 368.450 | 3.2× |

**CUDA breakeven ≈ 100,000 rows.** Auto-backend default `_CUDA_AUTO_MIN_ROWS` set to
`100_000` (was `1_000_000`) to match this breakeven — `backend="auto"` now picks CUDA
across the full 100k+ GPU-win range. Override per-call via `cuda_min_rows` or config
`greeks_cuda_min_rows` for slower GPUs / fast many-core CPUs. Covered by
`test_greeks.py::TestBatchGreeksLevel3.test_auto_default_threshold_is_100k`.
